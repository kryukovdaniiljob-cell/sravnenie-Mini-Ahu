# Логика сервиса сравнения — приточные и приточно-вытяжные установки

Документ описывает архитектуру данных, разделение на два типа установок, новую
методику присвоения баллов (0–100) и план изменений по файлам. Составлен до начала
реализации (по запросу: «продумай и пропиши логику и промты»).

---

## 1. Цель доработки

1. Логотип **SHUFT** в левом верхнем углу.
2. Два вида установок: **приточные** и **приточно-вытяжные (ПВУ)**, с чётким
   разграничением в интерфейсе — сравнивать приточные отдельно от ПВУ и наоборот.
3. Новая методика скоринга (две методики — для ПВУ и для приточных).
4. Корректный перенос полных данных из двух Excel-файлов.
5. Улучшения UX/UI.

## 3. Извлечение числовых величин (парсеры)

Грунтуется на реально встречающихся форматах из обоих файлов.

- **Расход** `flow_max` = максимальное целое из E. `«90-580»→580`, `«650»→650`.
- **Цена** `price_num` = целое из B, убрать пробелы/валюту. `«449500 ₽»→449500`.
- **Толщина** `thickness` = из G взять все целые ≥ 50, вернуть **минимальное**.
  `«968×760×568»→568`, `«660×706×280»→280`, `«н/д»→null`. Разделители `× х x *`.
- **Шум** `noise_max` = максимальное целое из L в диапазоне 10…90. `«30-47»→47`,
  `«57-70»→70`, `«н/д»→null`.
- **КПД** `eff` (только ПВУ) = первое число из шаблона `NN%` в U. `«до 89%»→89`,
  `«78%»→78`. У приточных U = `нет`/`н/д` → не используется.
- **Класс фильтра** `filter_rank` = по строке Q максимальный ранг по лестнице (см. 3.1).
- **Функции** `func_count` = число ячеек со значением **ровно «да»** среди
  {Wi-Fi (AJ), VAV (AL), Влажность (AM), CO₂ (AN)}. Диапазон 0…4.
  ⚠️ `опция` и `н/д` **не считаются** (строгое соответствие «да»).

### 3.1 Лестница рангов класса фильтрации

```
H14=18 · H13=17 · H11=15 · HEPA=15 · E12=14 · E11=13 · EPA=12 · E10=12 ·
EU9=11 · F9=11 · F8=10 · EU7=9 · F7=9 · F6=8 · EU5=7 · F5=7 · M6=6 · M5=6 ·
EU4=4 · G4=4 · EU3=3 · G3=3 · G2=2 · ПЫЛЕВОЙ=2 · G1=1
```

Алгоритм: Q привести к ВЕРХНЕМУ регистру, убрать пробелы; для каждого ключа лестницы
проверить вхождение подстрокой; вернуть **максимальный** найденный ранг; нет совпадения
→ нет данных (`null`).

Проверка на реальных значениях:
`«G4/HEPA (H11)»→15`, `«EPA E11»→13`, `«G4/F5/F7/F9»→7(F5)…max F9=11`→`11`,
`«пылевой»→2`, `«Префильтр высокой плотности»→null (нет данных)`, `«G4»→4`, `«EU5»→7`.

## 5. Баллы по параметрам (каждый 0–100)

Баллы считаются **внутри своего типа** (приточные — отдельно от ПВУ): сегментные
границы и глобальные min/max берутся по выборке этого типа.

### А) Сегментные (нормировка ВНУТРИ сегмента, МЕНЬШЕ = ЛУЧШЕ): Цена, Толщина, Шум
```
балл = 100 × (Макс_сегмента − значение) / (Макс_сег − Мин_сег)
```
min/max считаются по моделям сегмента, у которых параметр есть (не «н/д»).

### Б) Абсолютные (нормировка ГЛОБАЛЬНО по типу, БОЛЬШЕ = ЛУЧШЕ): Фильтр, Функции, КПД
```
Фильтр и КПД: балл = 100 × (значение − Мин_глоб) / (Макс_глоб − Мин_глоб)
Функции:      балл = значение / 4 × 100        (0→0, 1→25, 2→50, 3→75, 4→100)
```

### Правила краёв (для всех параметров)
- **Нет данных (н/д) → балл = 0** (штраф за неполноту, НЕ пропускать).
- **Макс = Мин (нет разброса) → балл = 50** (нейтрально, НЕ 100).
- Итог зажимается в [0, 100].

## 6. Итоговый балл

Среднее арифметическое **всех** параметров (равные веса), пропуски входят как 0.

- **ПВУ (6 параметров):** `ИТОГ = (Цена + Толщина + Шум + Фильтр + Функции + КПД) / 6`
- **Приточные (5 параметров):** `ИТОГ = (Цена + Толщина + Шум + Фильтр + Функции) / 5`

Чем больше пропусков — тем ниже ИТОГ (неполные позиции не завышаются).

**Эталон сегмента** = позиция с максимальным ИТОГ внутри сегмента (бейдж в карточке).
Ранжирование: по убыванию ИТОГ (внутри сегмента — честное сравнение однотипных по
размеру; глобально по типу — общий рейтинг).

### Оговорки
- Шум измеряется производителями по-разному (L wA у SHUFT/Breezart; звуковое давление
  у Turkov). Балл шума приблизительный, вес 1/6 (ПВУ) или 1/5 (приточные).
- КПД обязателен для ПВУ (рекуперативные установки) и отсутствует у приточных.
- Веса всех параметров равные.

## 7. Архитектура приложения и UX

### 7.1 Переключатель типа (главный элемент навигации)
Сегмент-контрол **«Приточные | Приточно-вытяжные»** вверху. Активный тип управляет:
каталогом, фильтрами, таблицей сравнения, справочником и калькулятором. Состояние —
в URL (`?type=supply|pvu`). Выбор моделей хранится **отдельно для каждого типа**;
сравнение — только в пределах одного типа (баллы типов несопоставимы — разное число
параметров и разные сегменты).

### 7.2 Фильтр по сегменту (новое)
Так как методика сегментная, добавляется фильтр «Сегмент (класс по расходу)» —
позволяет сравнивать модели одного размерного класса. Бейдж сегмента — в карточке и
в сравнении.

### 7.3 Карточка модели
Бренд (Rusklimat → **SHUFT**, подсветка «наш бренд»), наименование, балл 0–100 со
звёздами, бейдж сегмента (+«эталон» для лидера сегмента), ключевые характеристики
(расход, шум, фильтр), цена.

### 7.4 Сравнение
В пределах активного типа. Добавляется:
- строка «Оценка (0–100)»;
- раскрываемый **«Разбор оценки»** — под-баллы по 6/5 параметрам (полоски);
- строка «Сегмент»;
- для приточных секция «Рекуперация»/КПД скрывается (нерелевантна).

### 7.5 Справочник и калькулятор
Переписываются под новую методику и зависят от активного типа (6 vs 5 параметров,
сегментные vs абсолютные, правила краёв, лестница фильтра, таблицы сегментов).
Старый переключатель «статический/динамический» удаляется (методика всегда
относительна выборке типа).

## 10. Подбор аналога (SHUFT ⇄ конкуренты)

### 10.1 Задача
Пользователь видит модель конкурента (например, Turkov) и хочет эквивалент от
SHUFT — или наоборот: по модели SHUFT увидеть, какие модели конкурентов она
закрывает. Результат должен быть объясним (видно, по каким параметрам совпадение)
и сразу сравним (кнопка «Сравнить» ведёт в существующую таблицу сравнения).

### 10.2 Кандидаты (жёсткие условия)
1. **Тот же тип установки** (`supply`/`pvu`) — между типами аналоги не ищутся.
2. **Направление по бренду:** исходная НЕ SHUFT → кандидаты только SHUFT;
   исходная SHUFT → кандидаты все остальные бренды.
3. **Расход обязателен** у обеих сторон; окно кандидатов: |Fа − Fо| / Fо ≤ **0.40**.
   Проверено на данных: в окне ±40% SHUFT-кандидат есть у 180/182 приточных и
   216/241 ПВУ конкурентов; обратно — 55/55 и 102/106.
4. Нагреватель — строгость по решению 10.6.

### 10.3 Балл соответствия (0–100)
Взвешенная сумма мер близости. Цена в балл **не входит** — она результат подбора
(Δ цены показывается в каждой строке результата). Отсутствующие данные (у любой
из сторон) исключают параметр, веса перенормируются; в результате показывается
«совпадение по N из M параметров». Лишние функции аналога не штрафуются.

| Параметр | Вес ПУ | Вес ПВУ | Мера близости (0–100) |
|---|---|---|---|
| Расход | 40 | 35 | 100 × (1 − (|ΔF|/Fо)/0.40): 0% → 100, 40% → 0 |
| Нагреватель | 15 | 10 | тип совпал → 100 («вода+эл.» покрывает оба; «нет»+«нет» → 100), иначе 0 |
| Функции | 15 | 10 | покрытие набора оригинала: k из m функций → k/m × 100; m=0 → 100 |
| Шум | 10 | 10 | |Δ| ≤ 2 дБ → 100, линейно до 0 при 15 дБ |
| Толщина | 10 | 10 | 100 × (1 − (|Δt|/tо)/0.50): 0% → 100, 50% → 0 |
| Фильтрация | 10 | 10 | ранг аналога ≥ оригинала → 100; ниже → 100 − 20 × Δрангов (мин 0) |
| КПД (только ПВУ) | — | 15 | 100 × (1 − (|Δη|/ηо)/0.30): 0% → 100, 30% → 0 |

Суммы весов: ПУ = 40+15+15+10+10+10 = 100; ПВУ = 35+10+10+10+10+10+15 = 100.

### 10.4 Ранжирование и вывод
- Сортировка по убыванию балла соответствия; при равенстве выше модель с большим
  рейтингом 0–100 (предлагаем лучший из равных аналогов).
- Показываются до **5** кандидатов с баллом ≥ **50**.
- Категории: ≥85 «точный аналог» · 70–84 «близкий» · 50–69 «частичный».
- Пусто → «Аналогов не найдено» с причиной (нет моделей в окне расхода / нет расхода).

### 10.5 UI/UX (не трогает существующую логику)
- Кнопка на карточке: у конкурентов «⇄ Аналог от SHUFT», у SHUFT «⇄ Аналоги конкурентов».
- **Модальное окно**: сверху оригинал (компактно: расход, нагреватель, шум, цена),
  ниже список кандидатов — % соответствия, «по N из M параметров», ключевые
  параметры, Δ цены (зелёным, если аналог дешевле); у каждого кнопка «Сравнить»
  (выбор = [оригинал, аналог] → скролл к таблице), сверху «Сравнить топ-3»
  (оригинал + 3 лучших = 4 слота — максимум сравнения).
- Закрытие: ×, клик по фону, Esc. Мобайл — полноэкранный лист.
- Подбор — надстройка: отдельный модуль `analog.js`, движок рейтинга и сравнение
  не меняются; карточка получает только дополнительную кнопку.

### 10.6 Принятые решения (подтверждены 2026-07-02)
1. **Нагреватель — гибрид:** по умолчанию показываются только аналоги с совместимым
   типом нагревателя; в окне переключатель «показать и с другим нагревателем»
   (несовпадение тогда снижает балл через вес). «н/д» не исключает кандидата —
   параметр просто выпадает из балла.
2. **Охват — только SHUFT ⇄ конкуренты** (конкурент → SHUFT; SHUFT → конкуренты).
3. **«Сравнить» заменяет** текущий выбор на [оригинал, аналог].
