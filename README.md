# SHUFT · Сравнение приточных и приточно-вытяжных установок

Онлайн-сервис сравнения вентиляционных установок в минималистичном стиле. Два вида
установок — **приточные** и **приточно-вытяжные (ПВУ)** — с отдельным рейтингом
0–100 по размерным сегментам.

**Демо:** GitHub Pages → https://kryukovdaniiljob-cell.github.io/dorabotka_sravnenie/

## Возможности
- Переключатель типа **«Приточные | Приточно-вытяжные»** — каталог, фильтры,
  сравнение, справочник и калькулятор работают в пределах выбранного типа.
- Поиск, фильтры: бренд, **сегмент (размерный класс по расходу)**, цена, наличие
  клапана/фильтра, «только эталоны сегментов».
- Балл **0–100** в карточке (со звёздами), бейдж сегмента и «★ эталон сегмента».
- Выбор до **4 моделей** (отдельно для каждого типа), сравнение бок о бок,
  **«Разбор оценки»** — под-баллы по 6/5 параметрам.
- Приоритетные параметры сверху (расход, шум, фильтр), остальное — секциями.
  Секция «Рекуперация» показывается только для ПВУ.
- Бренд SHUFT (Rusklimat) подсвечивается как «наш бренд».
- **Подбор аналога** (кнопка «⇄» на карточке): конкурент → аналог SHUFT и наоборот.
  Балл соответствия 0–100 по расходу (якорь ±40%), нагревателю, функциям, шуму,
  толщине, фильтрации (+КПД для ПВУ); Δ цены в каждой строке; по умолчанию только
  совместимый нагреватель (переключатель «показать и с другим»); «Сравнить» ведёт
  пару [оригинал, аналог] в таблицу сравнения. Методика — LOGIC.md, раздел 10.
- **Аргументы продажи**: если в сравнении есть SHUFT и конкурент — над таблицей
  автоматически формируются готовые аргументы («дешевле на…», «тише на…», «есть
  Wi-Fi — у конкурента нет») с кнопкой «Скопировать» для письма клиенту; мелкой
  строкой — слабые места для подготовки к возражениям. Методика — LOGIC.md, §11.
- Прилипающая шапка, подсветка различий, адаптив, состояние в URL (`?type=…&c=…`).

## Методика оценки (0–100)
Балл считается **внутри своего типа**: сегментные параметры (цена, толщина, шум)
нормируются внутри размерного сегмента (меньше = лучше), абсолютные (фильтрация,
функции, КПД) — глобально по типу (больше = лучше). Нет данных → 0, нет разброса →
50. Итог — среднее всех параметров (ПВУ: 6 с КПД, приточные: 5 без КПД). Эталон
сегмента — модель с максимальным баллом в сегменте. Полное описание — [LOGIC.md](LOGIC.md).

## Данные
- `data.json` — 347 ПВУ + 237 приточных (VentMachine, Minibox, Breezart, Turkov,
  Horynize, SHUFT/Rusklimat). Источник — два Excel-файла конкурентного анализа
  (лист «Данные», 41 столбец A–AO).
- Сборка: `python tools/build_data.py` (требует `openpyxl` и оба `.xlsx` рядом).
  Скрипт парсит все поля, считает сегменты и печатает валидацию методики.
- «н/д» = параметр не опубликован. Цены и балл шума — справочные.

## Технологии
Чистые HTML + CSS + JavaScript, без зависимостей и сборки. `rating.js` — единый
движок оценки (каталог, справочник, калькулятор). Хостинг — статический (GitHub Pages,
автодеплой через `.github/workflows/deploy-pages.yml`).
